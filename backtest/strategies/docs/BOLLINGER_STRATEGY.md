# 볼린저 밴드 전략 V2 — 승율 개선 가이드

## 전략 개요

볼린저 밴드(Bollinger Bands) 기반의 평균회귀 전략. 가격이 통계적 상/하한 밖으로 벗어난 후 평균으로 복귀하는 것을 노린다.

**파일**: `backtest/strategies/bollinger.py`  
**클래스**: `BollingerBandStrategy`

---

## V1 → V2 개선사항

| 항목 | V1 (기존) | V2 (개선) |
|------|----------|----------|
| 진입 방식 | 하단 밴드 터치 즉시 매수 | **하단 밴드 아래 → 위 복귀 시 매수** (반등 확인) |
| 추세 필터 | 없음 | **MA50 필터** (상승 추세에서만 진입) |
| 밴드폭 필터 | 없음 | **최소 밴드폭** 조건 (횡보 스퀴즈 방지) |
| 손절 | 없음 | **ATR 트레일링 스톱** |
| 청산 옵션 | 상단 밴드 도달 시만 | **중선(MA) 청산 옵션** 추가 |
| 포지션 동기화 | 없음 | **외부 청산 반영** |

---

## 핵심 개선: 확인 진입

### 기존 방식의 문제
```
가격 < 하단 밴드 → 즉시 매수
```
- 강한 하락추세에서 가격이 하단 밴드를 따라 계속 내려감
- 하단 밴드가 함께 내려가므로 신호가 계속 발생 → 손실 누적

### 개선된 방식
```
직전 가격 < 하단 밴드 AND 현재 가격 >= 하단 밴드 → 매수 (반등 확인)
```
- 하단 밴드 아래로 내려갔다가 **다시 위로 올라오는 봉**에서 진입
- 실제 반등이 시작되는 시점 확인 → 허위 신호 대폭 감소

### %B 개념 활용
```
%B = (가격 - 하단 밴드) / (상단 밴드 - 하단 밴드)
%B < 0 → 하단 밴드 아래 (과매도)
%B > 1 → 상단 밴드 위 (과매수)
```
**내부적으로 was_below_lower 플래그로 추적**

## 파라미터

| 파라미터 | 기본값 | 설명 |
|---------|--------|------|
| `window` | 20 | 볼린저 밴드 이동평균 기간 |
| `num_std` | 2.0 | 표준편차 배수 (밴드 너비) |
| `invest_pct` | 1.0 | 투자 비율 |
| `trend_window` | 50 | 추세 필터 MA 기간 (0=비활성화) |
| `trail_mult` | 2.0 | ATR 트레일링 스톱 배수 |
| `exit_at_mid` | False | True 시 중선(MA) 도달에서 청산 (빠른 이익 실현) |
| `min_bandwidth` | 0.03 | 최소 밴드폭 비율 (상하단 간격/MA, 0=비활성화) |
| `market_df` | None | 시장 레짐 필터용 데이터 |

---

## 진입 조건

**모두 충족해야 매수:**

1. **확인 진입**: `직전 가격 < 하단 밴드` AND `현재 가격 >= 하단 밴드`
2. **추세 필터**: `가격 > MA(trend_window=50)` (하락추세 물타기 방지)
3. **밴드폭**: `(상단-하단)/MA >= min_bandwidth(0.03)` (3% 이상 변동성)
4. **레짐 필터**: `시장 > MA200` (market_df 제공 시)

---

## 청산 조건

**하나라도 충족 시 매도:**

1. `가격 > 상단 밴드` — 이익 실현
2. `exit_at_mid=True` AND `가격 >= MA(window)` — 빠른 중선 청산
3. `가격 < MA(trend_window)` — 추세 이탈 방어
4. ATR 트레일링 스톱 트리거
5. 시장 레짐 이탈

---

## 밴드폭 필터 (Bandwidth) 의미

```
밴드폭 = (상단 밴드 - 하단 밴드) / MA

밴드폭이 좁을 때 = 시장이 안정, 변동성 낮음
 → 볼린저 밴드 신호가 의미 없음 (반등폭 작음)

밴드폭이 넓을 때 = 변동성 높음
 → 유의미한 과매도/과매수 발생 가능 → 진입 허용
```

**min_bandwidth=0.03** → 밴드폭이 MA의 3% 미만이면 진입 안함

---

## exit_at_mid 옵션

```
False (기본): 상단 밴드 도달 시 청산 → 큰 수익 추구, 보유 기간 길다
True (빠른 청산): MA(중선) 도달 시 청산 → 승율 높지만 수익 작다
```

**승율 개선에는 exit_at_mid=True가 유리** (더 자주 이익 실현)

## 사용 예시

```python
from backtest.strategies import BollingerBandStrategy

# 승율 중시 설정 (빠른 청산 + 추세 필터)
strategy = BollingerBandStrategy(
    ticker="005930",
    window=20,
    num_std=2.0,
    trend_window=50,      # MA50 추세 필터
    trail_mult=2.0,       # ATR × 2 트레일링 스톱
    exit_at_mid=True,     # 중선 도달 시 청산 (승율 우선)
    min_bandwidth=0.03,   # 밴드폭 3% 이상
    invest_pct=1.0,
)

# 수익 중시 설정 (상단 밴드까지 보유)
strategy = BollingerBandStrategy(
    ticker="005930",
    window=20,
    num_std=2.0,
    trend_window=50,
    trail_mult=2.5,
    exit_at_mid=False,    # 상단 밴드 도달까지 보유
    min_bandwidth=0.03,
    invest_pct=0.7,
)
```

## 적합한 종목

- **박스권 등락** 종목 (평균 회귀 특성 강한 종목)
- 안정적 우상향하는 대형주 (추세 필터 적용 시)
- 변동성이 일정 수준 유지되는 종목

---

## 비적합 상황

- 강한 단방향 하락추세 (추세 필터로 1차 방어 가능)
- 밴드폭이 극히 좁은 횡보기 (min_bandwidth 필터로 방어)
- 갭하락 후 반등 패턴 (T+1 진입으로 갭을 놓칠 수 있음)
